# Лабораторна робота 1. Розробка швидкого прототипу веб-застосунку на Replit

## Мета роботи
Створення функціонального веб-застосунку з використанням сервісу Replit та інструментів штучного інтелекту для прискорення розробки програмного забезпечення.


## Теоретичні відомості

**Replit** — це хмарна інтегрована середа розробки (IDE), яка дозволяє створювати, редагувати та запускати код безпосередньо в браузері. Основні переваги:
- Миттєвий доступ без установки додаткового ПЗ
- Вбудована підтримка більше 50 мов програмування
- Автоматичне управління залежностями
- Інтеграція з GitHub та іншими сервісами
- Можливість миттєвого розгортання застосунків

**Flask** — це мікро веб-фреймворк для Python, який надає базові інструменти для створення веб-застосунків:
- Мінімалістичний підхід до веб-розробки
- Гнучкість у виборі компонентів
- Легкість вивчення та використання
- Широка екосистема розширень

**Prompt Engineering** — це процес створення ефективних запитів до систем штучного інтелекту для отримання бажаних результатів. Ключові принципи:
- Чіткість та конкретність завдання
- Надання контексту та прикладів
- Поетапна декомпозиція складних задач
- Перевірка та уточнення результатів

## Завдання за рівнями складності

### Базовий рівень (оцінка 6-7 балів)
Створити простий веб-застосунок з наступним функціоналом:
- Головна сторінка з описом проєкту
- Форма для введення даних (мінімум 3 поля)
- Збереження даних у файл або просту базу даних SQLite
- Відображення збережених даних у вигляді списку
- Базове CSS-оформлення

**Приклади тем:** Список завдань (To-Do), Книга відгуків, Каталог товарів

### Середній рівень (оцінка 8-9 балів)
Створити веб-застосунок з розширеним функціоналом:
- Все з базового рівня
- RESTful API з мінімум 4 ендпоінтами (GET, POST, PUT, DELETE)
- Робота з реляційною базою даних (SQLite/PostgreSQL)
- Валідація даних на сервері
- Обробка помилок та відповідні HTTP-коди
- Адаптивний дизайн
- Використання JavaScript для динамічної взаємодії

**Приклади тем:** Система управління бібліотекою, Блог з коментарями, Інтернет-магазин

### Високий рівень (оцінка 10 балів)
Створити повнофункціональний веб-застосунок:
- Все з середнього рівня
- Система аутентифікації користувачів
- Розподіл ролей (адміністратор, звичайний користувач)
- Інтеграція з зовнішнім API
- Документація API (Swagger/OpenAPI)
- Тести для основних функцій
- Docker-контейнеризація
- Професійний UI/UX дизайн

**Приклади тем:** CRM-система, Соціальна мережа, Платформа для навчання

## Хід виконання роботи

### Крок 1: Підготовка робочого середовища
1. Створіть обліковий запис на [replit.com](https://replit.com)
2. Створіть новий проєкт (Python/Flask)
3. Ознайомтеся з інтерфейсом Replit

### Крок 2: Планування проєкту
1. Оберіть тему вашого веб-застосунку
2. Визначте основний функціонал
3. Створіть схему бази даних
4. Сплануйте API-ендпоінти

### Крок 3: Використання ШІ для генерації коду

**Приклад промпту для Replit AI:**

```
Створи веб-застосунок на Flask для управління особистою колекцією книг з наступними вимогами:

1. Структура проєкту:
   - app.py (головний файл)
   - templates/ (HTML шаблони)
   - static/ (CSS, JS файли)
   - database.db (SQLite база даних)

2. База даних (таблиця books):
   - id (унікальний ідентифікатор)
   - title (назва книги)
   - author (автор)
   - year (рік видання)
   - genre (жанр)
   - status (прочитана/не прочитана)
   - created_at (дата додавання)

3. Функціонал:
   - Головна сторінка зі списком усіх книг
   - Сторінка додавання нової книги
   - Можливість позначити книгу як прочитану
   - Видалення книги з колекції
   - Пошук книг за назвою або автором
   - API ендпоінти для всіх операцій

4. Дизайн:
   - Використай Bootstrap для стилізації
   - Адаптивний дизайн
   - Інтуїтивний інтерфейс

5. API ендпоінти:
   - GET /api/books - отримати всі книги
   - POST /api/books - додати нову книгу
   - PUT /api/books/<id> - оновити статус книги
   - DELETE /api/books/<id> - видалити книгу

Додай валідацію даних, обробку помилок та базове логування. Код повинен бути готовим до запуску.
```

**Поради щодо промптів:**
- Будьте максимально конкретними у вимогах
- Вказуйте бажану структуру проєкту
- Описуйте схему бази даних детально
- Згадайте про обробку помилок та валідацію
- Просіть про коментарі у коді для кращого розуміння

### Крок 4: Тестування та налагодження
1. Запустіть проєкт у Replit
2. Протестуйте всі функції веб-застосунку
3. Перевірте роботу API через браузер або Postman
4. Виправте знайдені помилки

### Крок 5: Документування проєкту
1. Додайте README.md з описом проєкту
2. Створіть скріншоти інтерфейсу
3. Документуйте API ендпоінти

### Крок 6: Підготовка до здачі
1. Переконайтеся, що проєкт запускається без помилок
2. Створіть посилання для доступу до застосунку
3. Підготуйте звіт за запропонованим шаблоном

## Критерії оцінювання

| **Документація** | Базовий README | Детальний опис | API документація |

- Використання сучасних технологій (WebSockets, API інтеграції)
- Креативність у реалізації
- Высока якість коду та архітектури
- Додаткові функції поза вимогами
